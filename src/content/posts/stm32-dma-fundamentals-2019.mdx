DMA (Direct Memory Access) allows STM32 peripherals to transfer data directly to/from memory without CPU intervention. For high-throughput peripherals (SPI at 42 Mbps, ADC at 200 KS/s), DMA isn't optional -- the CPU simply can't service interrupts fast enough. DMA frees the CPU for computation while hardware handles data movement.

<Callout type="warning" title="High-Speed Peripherals Require DMA">
  SPI at maximum clock (42 MHz on APB2) generates a byte every ~4 us. An interrupt handler takes ~30 cycles minimum (entry + read DR + store + exit). At 168 MHz, that's 0.18 us per interrupt -- feasible per byte, but leaves only 3.8 us for everything else. Add ADC and UART, and the CPU is 100% saturated servicing interrupts. DMA eliminates this overhead entirely.
</Callout>

Each DMA stream connects a peripheral to memory (or memory to memory) with configurable transfer width, direction, and addressing mode.

## Core Configuration Pattern

Every DMA setup follows the same pattern: source address, destination address, transfer count, and trigger source:

```c
// DMA2 Stream 0, Channel 0: ADC1 -> memory buffer
void setup_adc_dma(uint16_t *buffer, uint32_t count) {
    RCC->AHB1ENR |= RCC_AHB1ENR_DMA2EN;  // Enable DMA2 clock

    DMA2_Stream0->CR = 0;                  // Reset stream
    while (DMA2_Stream0->CR & DMA_SxCR_EN); // Wait for disable

    DMA2_Stream0->PAR  = (uint32_t)&ADC1->DR;  // Source: ADC data register
    DMA2_Stream0->M0AR = (uint32_t)buffer;      // Destination: RAM buffer
    DMA2_Stream0->NDTR = count;                  // Number of transfers

    DMA2_Stream0->CR = DMA_SxCR_CHSEL_0   // Channel 0 (ADC1)
        | DMA_SxCR_MSIZE_0                 // Memory: 16-bit (half-word)
        | DMA_SxCR_PSIZE_0                 // Peripheral: 16-bit
        | DMA_SxCR_MINC                    // Memory address increment
        | DMA_SxCR_CIRC                    // Circular mode
        | DMA_SxCR_TCIE                    // Transfer complete interrupt
        | DMA_SxCR_EN;                     // Enable stream
}
```

<Callout type="tip" title="Circular Mode for Continuous Acquisition">
  Circular mode (`DMA_SxCR_CIRC`) automatically restarts the transfer when the buffer is full -- essential for continuous ADC sampling and audio streaming. Without circular mode, you'd need to reconfigure DMA in the transfer-complete ISR, adding latency.
</Callout>

## Double Buffering

DMA double buffering uses two memory buffers: while DMA fills one, the CPU processes the other:

```c
uint16_t buffer_a[BLOCK_SIZE];
uint16_t buffer_b[BLOCK_SIZE];

void setup_double_buffer_dma(void) {
    DMA2_Stream0->CR |= DMA_SxCR_DBM;           // Enable double buffer mode
    DMA2_Stream0->M0AR = (uint32_t)buffer_a;      // Buffer A
    DMA2_Stream0->M1AR = (uint32_t)buffer_b;      // Buffer B
    // DMA automatically alternates between buffers
}

void DMA2_Stream0_IRQHandler(void) {
    if (DMA2->LISR & DMA_LISR_TCIF0) {
        DMA2->LIFCR = DMA_LIFCR_CTCIF0;  // Clear interrupt flag

        // Which buffer just completed?
        if (DMA2_Stream0->CR & DMA_SxCR_CT) {
            process_buffer(buffer_a);  // Buffer A complete, process it
        } else {
            process_buffer(buffer_b);  // Buffer B complete, process it
        }
    }
}
```

<Callout type="info" title="Half-Transfer Interrupt Pattern">
  For continuous ADC sampling, use the half-transfer interrupt (HTIE) in addition to the transfer-complete interrupt (TCIE). Process the first half of the buffer in the HT ISR while DMA fills the second half, and vice versa. This is simpler than double buffering and works with a single buffer -- effectively ping-pong processing within one allocation.
</Callout>

## Conclusion

DMA is essential for any STM32 application using high-speed peripherals (SPI over 1 MHz, ADC continuous, I2S audio). It reduces CPU load from 85%+ to under 3% for SPI transfers, enabling the CPU to focus on computation. The key patterns are: circular mode for continuous acquisition, double buffering for zero-gap streaming, and FIFO mode for burst efficiency. Start with single-buffer circular DMA for ADC/UART, and upgrade to double-buffer when you need guaranteed zero-gap throughput.
