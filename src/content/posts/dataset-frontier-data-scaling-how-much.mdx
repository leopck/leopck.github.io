The Chinchilla scaling law (Hoffmann et al., 2022) established that compute-optimal training uses approximately 20 tokens per parameter. A 70B model should be trained on 1.4T tokens. Llama 3 trains 70B models on 15T tokens -- 10x the Chinchilla-optimal amount. Why? Because inference cost dominates training cost, and over-trained models perform better per inference FLOP.

But this creates a new problem: where do you get 15 trillion high-quality tokens? The entire English internet, aggressively filtered, yields approximately 5-10 trillion tokens. We are approaching the "data wall" -- the point where there is not enough naturally occurring high-quality text to train the next generation of models.

This post covers the data scaling law, the economics of over-training, the data wall, and the strategies for breaking through it.

---

## 1. The Chinchilla Scaling Law

### The Original Result

Hoffmann et al. trained over 400 language models with different combinations of model size $N$ and training tokens $D$, holding total compute $C \propto 6ND$ constant. They fit the loss function:

$$L(N, D) = \frac{A}{N^\alpha} + \frac{B}{D^\beta} + E$$

where $A = 406.4$, $B = 410.7$, $\alpha = 0.34$, $\beta = 0.28$, $E = 1.69$.

The compute-optimal allocation minimizes $L$ for a given compute budget $C$:

$$N_{\text{opt}} \propto C^{0.50}, \quad D_{\text{opt}} \propto C^{0.50}$$

This gives the "20x rule": $D_{\text{opt}} \approx 20 \times N$. For a 70B model: $D_{\text{opt}} = 20 \times 70\text{B} = 1.4\text{T}$ tokens.

### Why Over-Train? The Inference Economics

Chinchilla optimizes for minimum training loss at a given training compute budget. But in production, the dominant cost is inference, not training. Training happens once; inference happens billions of times.

A model trained for $2\times$ longer (same model size, $2\times$ more tokens) requires $2\times$ the training compute but zero additional inference compute. If the extra training improves the model enough to serve the same traffic with fewer inference calls (e.g., better one-shot accuracy, fewer retries), the total cost decreases.

```python
def compute_total_cost(
    model_params_B: float,
    training_tokens_T: float,
    training_flops_per_token: float = 6.0,
    gpu_flops_per_second: float = 1e15,  # H100 BF16
    gpu_cost_per_hour: float = 3.0,
    inference_queries_per_day: float = 1e7,
    inference_tokens_per_query: float = 500,
    deployment_days: int = 365,
    model_quality_factor: float = 1.0,  # Higher = fewer inference calls needed
):
    """Compare training cost vs. inference cost."""
    # Training cost
    total_training_flops = model_params_B * 1e9 * training_tokens_T * 1e12 * training_flops_per_token
    training_gpu_hours = total_training_flops / (gpu_flops_per_second * 3600)
    training_cost = training_gpu_hours * gpu_cost_per_hour

    # Inference cost (per day)
    effective_queries = inference_queries_per_day / model_quality_factor
    inference_flops_per_day = (
        effective_queries * inference_tokens_per_query *
        model_params_B * 1e9 * 2  # ~2 FLOPs per parameter per token for inference
    )
    inference_gpu_hours_per_day = inference_flops_per_day / (gpu_flops_per_second * 3600)
    inference_cost_per_day = inference_gpu_hours_per_day * gpu_cost_per_hour
    total_inference_cost = inference_cost_per_day * deployment_days

    return {
        "training_cost": training_cost,
        "inference_cost": total_inference_cost,
        "total_cost": training_cost + total_inference_cost,
        "training_fraction": training_cost / (training_cost + total_inference_cost),
    }

# Chinchilla-optimal: 70B model, 1.4T tokens
chinchilla = compute_total_cost(70, 1.4, model_quality_factor=1.0)

# Over-trained: 70B model, 15T tokens, 5% better quality
overtrained = compute_total_cost(70, 15, model_quality_factor=1.05)

# Training cost: $2.1M -> $22.5M (10x increase)
# Inference cost: $180M -> $171M (5% decrease due to better quality)
# Total cost: $182M -> $194M -- over-training costs more in this scenario
# But if quality improvement is 15%:
# Inference cost: $180M -> $156M, Total: $179M -> saves $3M
```

The breakeven quality improvement for over-training depends on the inference-to-training cost ratio. For high-traffic services, even a 2% quality improvement from over-training pays for itself.

### The Repetition Problem

When you run out of unique data, you repeat data. But repetition has diminishing returns and can cause memorization artifacts:

```python
import numpy as np

def model_loss_with_repetition(
    unique_tokens_T: float,
    total_tokens_T: float,
    alpha: float = 0.28,  # Chinchilla data exponent
    repetition_penalty: float = 0.15,
):
    """
    Model effective loss accounting for data repetition.

    Loss improves with more unique tokens (Chinchilla law).
    Repeated tokens provide diminishing value.
    """
    if total_tokens_T <= unique_tokens_T:
        # No repetition: standard scaling
        effective_tokens = total_tokens_T
    else:
        # Repetition: each additional epoch is worth less
        num_epochs = total_tokens_T / unique_tokens_T
        # Effective tokens grow logarithmically with epochs
        effective_tokens = unique_tokens_T * (
            1 + repetition_penalty * np.log(num_epochs)
        )

    # Chinchilla-style loss
    loss = 410.7 / (effective_tokens * 1e12) ** alpha + 1.69
    return loss, effective_tokens

# Compare: 5T unique tokens vs. 5T unique + 10T repeated
loss_5T, eff_5T = model_loss_with_repetition(5, 5)
loss_15T, eff_15T = model_loss_with_repetition(5, 15)

# 15T total tokens (5T unique + 10T repeated)
# Effective: ~5T * (1 + 0.15 * ln(3)) = 5T * 1.16 = 5.82T
# Only 16% more effective than 5T unique
```

<Callout type="warning" title="Repetition Has Hard Limits">
Muennighoff et al. (2023) showed that training on 4+ epochs of the same data causes measurable quality degradation. The model begins to memorize specific training examples rather than learning generalizable patterns. After 8 epochs, the degradation outweighs the benefit of additional training steps. This sets a hard limit on how far repetition alone can take you.
</Callout>

### Token Deduplication at Scale

Before counting available tokens, deduplication is essential. The raw web contains massive amounts of duplicated content:

```python
class TokenBudgetAnalyzer:
    """Analyze token budgets after deduplication."""

    def analyze_duplication(self, dataset_stats):
        """Compute effective unique tokens after dedup."""
        results = {}
        for source, stats in dataset_stats.items():
            raw_tokens = stats["raw_tokens_T"]
            # Typical dedup ratios by source
            dedup_ratios = {
                "common_crawl": 0.40,   # 60% is duplicate
                "code": 0.55,           # 45% duplicate
                "books": 0.85,          # 15% duplicate
                "academic": 0.90,       # 10% duplicate
                "wikipedia": 0.95,      # 5% duplicate
            }
            ratio = dedup_ratios.get(stats["source_type"], 0.50)
            unique = raw_tokens * ratio

            results[source] = {
                "raw_tokens_T": raw_tokens,
                "unique_tokens_T": unique,
                "duplication_rate": 1 - ratio,
            }

        total_raw = sum(r["raw_tokens_T"] for r in results.values())
        total_unique = sum(r["unique_tokens_T"] for r in results.values())

        return {
            "sources": results,
            "total_raw_T": total_raw,
            "total_unique_T": total_unique,
            "overall_dedup_rate": 1 - total_unique / total_raw,
        }
```

## 3. Breaking Through the Data Wall

### Strategy 1: Synthetic Data

Use a strong model to generate training data for the next model:

```python
class SyntheticDataGenerator:
    """Generate synthetic training data to augment natural data."""

    def __init__(self, teacher_model, tokenizer):
        self.teacher = teacher_model
        self.tokenizer = tokenizer

    def generate_continuation(self, seed_text, max_tokens=2048):
        """Generate a continuation from a seed text."""
        input_ids = self.tokenizer.encode(seed_text, return_tensors="pt").to("cuda")
        with torch.no_grad():
            output = self.teacher.generate(
                input_ids,
                max_new_tokens=max_tokens,
                temperature=0.8,
                top_p=0.95,
            )
        return self.tokenizer.decode(output[0], skip_special_tokens=True)

    def generate_textbook(self, topic, subtopics):
        """Generate textbook-style content on a topic."""
        seed = (
            f"# {topic}\n\n"
            f"## Introduction\n\n"
            f"This chapter covers {topic} in depth, including "
            f"{', '.join(subtopics)}.\n\n"
        )
        return self.generate_continuation(seed, max_tokens=4096)

    def generate_code(self, description, language="python"):
        """Generate code with documentation."""
        seed = (
            f"# {description}\n"
            f"# Language: {language}\n"
            f"# This implementation includes comprehensive documentation.\n\n"
        )
        return self.generate_continuation(seed, max_tokens=2048)

    def estimate_synthetic_budget(
        self,
        target_tokens_T: float,
        unique_natural_T: float,
        max_epochs: int = 4,
    ):
        """Estimate how much synthetic data is needed."""
        max_natural = unique_natural_T * max_epochs
        deficit = max(0, target_tokens_T - max_natural)

        # Synthetic data is ~70% as effective as natural data
        synthetic_efficiency = 0.70
        synthetic_needed = deficit / synthetic_efficiency

        return {
            "target_tokens_T": target_tokens_T,
            "natural_available_T": max_natural,
            "deficit_T": deficit,
            "synthetic_needed_T": synthetic_needed,
            "synthetic_fraction": synthetic_needed / target_tokens_T,
        }
```

### Strategy 2: Multilingual Data

Use non-English text to provide additional signal:

```python
class MultilingualDataMixer:
    """Mix multilingual data sources."""

    LANGUAGE_DATA = {
        "en": {"tokens_T": 6.5, "quality": 1.0},
        "zh": {"tokens_T": 3.0, "quality": 0.85},
        "de": {"tokens_T": 1.2, "quality": 0.90},
        "fr": {"tokens_T": 1.0, "quality": 0.90},
        "es": {"tokens_T": 0.9, "quality": 0.88},
        "ja": {"tokens_T": 0.8, "quality": 0.85},
        "ru": {"tokens_T": 0.7, "quality": 0.82},
        "ko": {"tokens_T": 0.4, "quality": 0.83},
        "pt": {"tokens_T": 0.5, "quality": 0.85},
        "other": {"tokens_T": 2.0, "quality": 0.70},
    }

    def compute_multilingual_budget(self, target_T: float, english_fraction: float = 0.60):
        """Compute data mix across languages."""
        english_target = target_T * english_fraction
        non_english_target = target_T * (1 - english_fraction)

        allocation = {"en": min(english_target, self.LANGUAGE_DATA["en"]["tokens_T"])}
        remaining = non_english_target

        # Distribute remaining proportionally to available data
        non_en_total = sum(
            v["tokens_T"] for k, v in self.LANGUAGE_DATA.items() if k != "en"
        )

        for lang, info in self.LANGUAGE_DATA.items():
            if lang == "en":
                continue
            proportion = info["tokens_T"] / non_en_total
            allocation[lang] = min(
                remaining * proportion * info["quality"],
                info["tokens_T"],
            )

        total_allocated = sum(allocation.values())
        return {
            "allocation": allocation,
            "total_T": total_allocated,
            "target_T": target_T,
            "coverage": total_allocated / target_T,
        }
```

## 4. Projecting Data Requirements

### Future Model Sizes and Token Needs

```python
def project_data_needs(years_ahead=5):
    """Project training data requirements for future models."""
    # Current: ~15T tokens for frontier models (2024)
    # Assume model size doubles every ~18 months
    # Assume over-training factor stays at ~200 tokens/param

    projections = []
    current_params_B = 400   # Llama 3.1 405B
    current_tokens_T = 15
    tokens_per_param = current_tokens_T * 1e12 / (current_params_B * 1e9)

    for year in range(years_ahead):
        # Model size growth: ~1.5x per year
        params_B = current_params_B * (1.5 ** year)
        # Token requirement grows with model size
        tokens_T = params_B * 1e9 * tokens_per_param / 1e12

        # Available data (growing slowly: ~20% per year from new web content)
        available_natural_T = 10 * (1.2 ** year)
        available_synthetic_T = 5 * (2.0 ** year)  # Synthetic grows faster
        total_available = available_natural_T + available_synthetic_T

        projections.append({
            "year": 2025 + year,
            "model_params_B": round(params_B),
            "tokens_needed_T": round(tokens_T, 1),
            "natural_available_T": round(available_natural_T, 1),
            "synthetic_available_T": round(available_synthetic_T, 1),
            "total_available_T": round(total_available, 1),
            "data_sufficient": total_available >= tokens_T,
        })

    return projections
```

## 5. Complete Data Budget Planner

```python
class DataBudgetPlanner:
    """
    Plan training data budget for a model training run.
    Accounts for natural data, synthetic data, repetition, and quality.
    """

    def __init__(self):
        self.multilingual = MultilingualDataMixer()

    def plan(
        self,
        model_params_B: float,
        target_tokens_T: float,
        max_epochs: int = 4,
        synthetic_fraction_max: float = 0.40,
        english_fraction: float = 0.65,
    ):
        """Create a complete data budget plan."""
        # Step 1: Multilingual allocation
        multilingual = self.multilingual.compute_multilingual_budget(
            target_tokens_T, english_fraction
        )

        # Step 2: Account for repetition
        unique_available = multilingual["total_T"]
        max_from_repetition = unique_available * max_epochs

        # Step 3: Synthetic data to fill the gap
        deficit = max(0, target_tokens_T - max_from_repetition)
        synthetic_needed = deficit / 0.70  # 70% efficiency

        # Cap synthetic data
        if synthetic_needed / target_tokens_T > synthetic_fraction_max:
            synthetic_needed = target_tokens_T * synthetic_fraction_max

        # Step 4: Compute effective tokens
        natural_effective = unique_available * (
            1 + 0.15 * np.log(max(1, min(max_epochs, target_tokens_T / unique_available)))
        )
        synthetic_effective = synthetic_needed * 0.70
        total_effective = natural_effective + synthetic_effective

        # Step 5: Chinchilla-optimal comparison
        chinchilla_optimal = model_params_B * 20 / 1000  # In trillions

        return {
            "model_params_B": model_params_B,
            "target_tokens_T": target_tokens_T,
            "chinchilla_optimal_T": chinchilla_optimal,
            "over_training_ratio": target_tokens_T / chinchilla_optimal,
            "data_sources": {
                "natural_unique_T": unique_available,
                "natural_with_repetition_T": max_from_repetition,
                "synthetic_T": synthetic_needed,
                "total_raw_T": max_from_repetition + synthetic_needed,
            },
            "effective_tokens_T": total_effective,
            "language_allocation": multilingual["allocation"],
            "synthetic_fraction": synthetic_needed / (max_from_repetition + synthetic_needed),
            "epochs_over_natural": target_tokens_T / unique_available,
            "feasible": (max_from_repetition + synthetic_needed) >= target_tokens_T,
        }
```
