Deep sleep on the ESP32 reduces power consumption from ~240 mA (active WiFi) to ~10 µA — a 24,000× reduction. But all SRAM contents are lost. The **RTC memory** (8 KB fast + 8 KB slow) survives deep sleep, providing a way to persist small amounts of state without flash writes. This is essential for battery-powered sensor nodes that wake periodically.

## Using RTC Memory

Variables placed in RTC memory persist across deep sleep cycles:

```c
#include "esp_sleep.h"

// RTC_DATA_ATTR places variable in RTC SLOW memory
RTC_DATA_ATTR static int boot_count = 0;
RTC_DATA_ATTR static float last_temperature = 0.0;
RTC_DATA_ATTR static uint32_t wifi_channel = 0;
RTC_DATA_ATTR static uint8_t wifi_bssid[6] = {0};

// RTC_NOINIT_ATTR: Same as above but not initialized on first boot
// Useful for detecting cold boot vs wake from deep sleep
RTC_NOINIT_ATTR static uint32_t magic_value;

void app_main() {
    boot_count++;

    if (magic_value != 0xDEADBEEF) {
        // Cold boot — initialize everything
        magic_value = 0xDEADBEEF;
        boot_count = 1;
    }

    printf("Boot #%d, last temp: %.1f°C\n", boot_count, last_temperature);
}
```

<Callout type="warning" title="8 KB Budget">
  RTC SLOW memory is only 8 KB total, shared between your variables and the ULP coprocessor program. A 1000-sample float buffer (4 KB) uses half the space. Plan your data structures carefully — use fixed-point integers instead of floats where possible (2 bytes vs 4 bytes per sample).
</Callout>

## Pattern: Sensor Data Logging

The most common pattern: wake from deep sleep, read sensor, store in RTC buffer, go back to sleep. When the buffer is full, wake WiFi and transmit:

```c
#define BUFFER_SIZE 200  // 200 readings before transmit
#define SLEEP_TIME_US (60 * 1000000)  // Wake every 60 seconds

RTC_DATA_ATTR static uint16_t sensor_buffer[BUFFER_SIZE];  // 400 bytes
RTC_DATA_ATTR static uint16_t buffer_index = 0;

void app_main() {
    // Read sensor (fast — no WiFi needed)
    uint16_t reading = read_temperature_fixed_point();
    sensor_buffer[buffer_index++] = reading;

    if (buffer_index >= BUFFER_SIZE) {
        // Buffer full — wake WiFi and transmit
        wifi_init_fast_reconnect();
        transmit_buffer(sensor_buffer, BUFFER_SIZE);
        buffer_index = 0;
    }

    // Back to deep sleep
    esp_sleep_enable_timer_wakeup(SLEEP_TIME_US);
    esp_deep_sleep_start();
}
```

## Pattern: WiFi Fast Reconnect

WiFi connection typically takes 2-5 seconds (scanning + DHCP). By caching the channel and BSSID in RTC memory, you skip the scan phase:

```c
RTC_DATA_ATTR static uint8_t saved_bssid[6];
RTC_DATA_ATTR static uint8_t saved_channel;
RTC_DATA_ATTR static bool has_saved_wifi;

void wifi_init_fast_reconnect() {
    wifi_config_t wifi_config = {
        .sta = {
            .ssid = "MyNetwork",
            .password = "MyPassword",
        },
    };

    if (has_saved_wifi) {
        // Skip scan — connect directly to saved AP
        memcpy(wifi_config.sta.bssid, saved_bssid, 6);
        wifi_config.sta.bssid_set = true;
        wifi_config.sta.channel = saved_channel;
        // Connects in ~300 ms instead of 2-5 seconds
    }

    esp_wifi_set_config(ESP_IF_WIFI_STA, &wifi_config);
    esp_wifi_connect();
}

void on_wifi_connected(wifi_event_sta_connected_t *event) {
    // Cache for next wake cycle
    memcpy(saved_bssid, event->bssid, 6);
    saved_channel = event->channel;
    has_saved_wifi = true;
}
```

<Callout type="tip" title="Maximize Your Buffer">
  Use `uint16_t` (2 bytes) instead of `float` (4 bytes) for sensor data by converting to fixed-point: `uint16_t temp = (uint16_t)(temperature * 100)`. This doubles your buffer capacity. For a 16-bit ADC reading, you lose nothing — the ADC output is already 16 bits.
</Callout>

RTC memory is the right choice for data that changes every wake cycle (sensor readings, counters, WiFi cache). Flash NVS is better for data that changes rarely (configuration, calibration) and needs to survive power loss.

## Conclusion

ESP32's 16 KB of RTC memory enables efficient battery-powered IoT designs: buffer sensor data across deep sleep cycles (15-23× power reduction vs per-reading WiFi), cache WiFi parameters for fast reconnect (7.5× energy reduction per connection), and maintain application state without flash wear. The 8 KB budget requires careful planning — use fixed-point representation and pack data structures tightly. For sensor nodes waking every 60 seconds, the combination of RTC buffering + fast WiFi reconnect can extend CR2032 battery life from days to months.
