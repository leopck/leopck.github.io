Attention dominates transformer inference time, and its O(N²) scaling makes it the primary bottleneck for long sequences. Before optimizing, we need to understand exactly where the cycles are spent.

## The Attention Computation Pipeline

Standard scaled dot-product attention involves five sequential operations. Profiling each individually reveals a clear picture:

Two operations dominate: QK^T (45%) and softmax (37%). The QK^T matmul is compute-bound — it's a large matrix multiplication that tensor cores handle well. But softmax is **memory-bandwidth-bound**: it requires reading the entire N×N score matrix, computing element-wise exponentials, normalizing, and writing back. This read-compute-write pattern is the fundamental bottleneck.

## Quadratic Scaling in Practice

The O(N²) scaling isn't just theoretical — it hits hard in practice. Doubling sequence length doesn't double attention time; it quadruples it.

## Memory Access Analysis

The memory access pattern reveals why attention is bandwidth-bound at scale. For each attention head with sequence length N and head dimension d:

The N×N score matrix is the culprit. It's read and written **three times** (store QK^T, load for softmax, store softmax result) before being used in the final matmul. For 32 heads at N=4096, total HBM traffic exceeds 4 GB per layer.

<Callout type="perf" title="The Key Insight">
  The N×N attention score matrix is a massive intermediate that exists only to be immediately consumed. It doesn't need to live in HBM at all — if we can compute and consume it in tiles that fit in SRAM, we eliminate 97% of memory traffic. This is exactly what FlashAttention does.
</Callout>

Standard attention operations have arithmetic intensity of 1-3 FLOP/byte — deep in memory-bound territory on modern GPUs. Fusing everything into a single kernel with tiled SRAM access pushes arithmetic intensity to ~89 FLOP/byte, crossing into compute-bound territory.

## Optimization Approaches

The analysis points to three complementary optimization strategies:

**Tiled/Fused Attention (FlashAttention):** Eliminate HBM traffic for the N×N intermediate by processing in SRAM-sized tiles. This is the most impactful optimization — it doesn't approximate, it computes the exact same result with 33× less memory traffic.

**Sparse Attention Patterns:** Reduce the effective N² computation by only attending to a subset of tokens. Local attention (window size W) reduces complexity from O(N²) to O(NW). The trade-off is model quality — sparse patterns lose long-range dependencies.

**Low-Rank Approximation:** If attention weights are approximately low-rank (rank r << N), you can compute Q_low K_low^T in O(N²r) instead of O(N²d). This trades approximation error for speed, and works best when attention patterns are smooth.

## Conclusion

Profiling attention reveals a clear hierarchy of bottlenecks. The QK^T and softmax operations consume 82% of time, and the root cause is the N×N intermediate matrix that requires 97% of HBM traffic. The optimization path is clear: eliminate that intermediate (FlashAttention), reduce its size (sparse attention), or compress it (low-rank). Profile first, optimize the actual bottleneck, measure the improvement.
