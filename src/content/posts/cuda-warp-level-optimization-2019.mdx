A warp is the fundamental execution unit on NVIDIA GPUs — 32 threads that execute in lockstep. Warp-level primitives (shuffle, vote, match) allow threads within a warp to communicate directly through registers, bypassing shared memory entirely. This is often 2-3× faster than the shared memory approach for small-data communication patterns.

## Warp Shuffle: Register-to-Register Communication

`__shfl_sync` and variants allow any thread in a warp to read any other thread's register value — no shared memory, no synchronization barrier:

```cpp
// Warp-level reduction using shuffle (32 elements, one per thread)
__device__ float warp_reduce_sum(float val) {
    for (int offset = 16; offset > 0; offset >>= 1) {
        val += __shfl_down_sync(0xFFFFFFFF, val, offset);
    }
    return val;  // Thread 0 has the final sum
}
```

Five `__shfl_down_sync` instructions replace what would be 5 shared memory writes + 5 reads + 5 `__syncthreads()`. The shuffle path is faster because register-to-register transfer has essentially zero latency (routed through the warp scheduler, not through SRAM banks).

The XOR variant is particularly elegant for reductions: `__shfl_xor_sync(0xFFFFFFFF, val, 1)` swaps between adjacent pairs, `__shfl_xor_sync(0xFFFFFFFF, val, 2)` swaps between pairs of pairs, and so on — implementing a butterfly reduction pattern.

```cpp
// Butterfly reduction (often faster than tree on some architectures)
__device__ float warp_reduce_butterfly(float val) {
    val += __shfl_xor_sync(0xFFFFFFFF, val, 16);
    val += __shfl_xor_sync(0xFFFFFFFF, val, 8);
    val += __shfl_xor_sync(0xFFFFFFFF, val, 4);
    val += __shfl_xor_sync(0xFFFFFFFF, val, 2);
    val += __shfl_xor_sync(0xFFFFFFFF, val, 1);
    return val;  // ALL threads have the sum (not just thread 0)
}
```

<Callout type="tip" title="Butterfly vs Tree">
  `__shfl_down_sync` (tree) gives the result only to thread 0. `__shfl_xor_sync` (butterfly) gives the result to ALL threads. If every thread needs the reduced value (common in normalization), butterfly saves a broadcast.
</Callout>

### Practical Example: Warp-Level Early Exit

```cpp
// Skip computation if entire warp has zero input
__global__ void sparse_process(float *data, float *output, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx >= n) return;

    float val = data[idx];

    // If ALL 32 threads in this warp have zero input, skip entirely
    if (__all_sync(0xFFFFFFFF, val == 0.0f)) {
        output[idx] = 0.0f;
        return;  // Entire warp exits — no divergence
    }

    // Expensive computation only for warps with non-zero data
    output[idx] = expensive_function(val);
}
```

This pattern is invaluable for sparse neural networks: if a warp's entire tile of activations is zero (common in ReLU networks), the warp skips computation entirely — not just individual threads.

## Warp-Level Prefix Scan

Prefix scan (cumulative sum) is a fundamental parallel primitive. Warp shuffle makes it efficient without shared memory:

```cpp
// Inclusive prefix sum within a warp (Hillis-Steele algorithm)
__device__ float warp_prefix_sum(float val) {
    float temp;
    temp = __shfl_up_sync(0xFFFFFFFF, val, 1);
    if (threadIdx.x % 32 >= 1) val += temp;

    temp = __shfl_up_sync(0xFFFFFFFF, val, 2);
    if (threadIdx.x % 32 >= 2) val += temp;

    temp = __shfl_up_sync(0xFFFFFFFF, val, 4);
    if (threadIdx.x % 32 >= 4) val += temp;

    temp = __shfl_up_sync(0xFFFFFFFF, val, 8);
    if (threadIdx.x % 32 >= 8) val += temp;

    temp = __shfl_up_sync(0xFFFFFFFF, val, 16);
    if (threadIdx.x % 32 >= 16) val += temp;

    return val;
}
```

## Block-Level Reductions Using Warps

For blocks larger than 32 threads, combine warp-level and shared memory approaches:

```cpp
// Two-level reduction: warp shuffle + shared memory
__device__ float block_reduce_sum(float val) {
    // Phase 1: Reduce within each warp using shuffle
    val = warp_reduce_sum(val);

    // Phase 2: First thread of each warp writes to shared memory
    __shared__ float warp_sums[32];  // Max 32 warps per block
    int warp_id = threadIdx.x / 32;
    int lane_id = threadIdx.x % 32;

    if (lane_id == 0) warp_sums[warp_id] = val;
    __syncthreads();

    // Phase 3: First warp reduces across warp sums
    if (warp_id == 0) {
        val = (lane_id < blockDim.x / 32) ? warp_sums[lane_id] : 0.0f;
        val = warp_reduce_sum(val);
    }
    return val;  // Thread 0 has block sum
}
```

This pattern uses shared memory only for 1 value per warp (not 1 per thread), minimizing shared memory usage and bank conflicts.

<Callout type="info" title="This Is What CUB/CUTLASS Use">
  NVIDIA's CUB library and CUTLASS GEMM kernels use exactly this two-level reduction pattern. It's the established best practice: warp shuffle for intra-warp, minimal shared memory for inter-warp.
</Callout>

## Conclusion

Warp-level primitives are the fastest communication mechanism on NVIDIA GPUs — register-to-register transfer with zero memory traffic. Use `__shfl_down_sync` for tree reductions (result in lane 0), `__shfl_xor_sync` for butterfly reductions (result in all lanes), and `__all_sync`/`__any_sync` for collective predicates. For block-level operations, the two-level approach (warp shuffle + minimal shared memory) is the gold standard used by CUB and CUTLASS. The performance difference is real: 2-3× faster than pure shared memory approaches for small reductions.
