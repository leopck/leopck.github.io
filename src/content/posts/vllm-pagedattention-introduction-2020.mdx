PagedAttention is vLLM's key innovation: apply virtual memory concepts from operating systems to GPU KV cache management. The result is near-zero memory waste, which translates directly into serving more concurrent requests and higher throughput.

## The Memory Fragmentation Problem

Traditional LLM serving allocates KV cache memory **contiguously and upfront** for each request at the maximum possible sequence length. This creates massive waste:

## PagedAttention: The OS Analogy

PagedAttention borrows directly from how operating systems manage virtual memory. Instead of allocating contiguous blocks, KV cache is divided into fixed-size **pages** (typically 16 tokens each). Each request maintains a **page table** mapping logical token positions to physical page locations.

<Callout type="perf" title="Why This Matters">
  With PagedAttention, a 128-token response consumes exactly 8 pages (128/16) of KV memory, not 128 pages (2048/16). Memory efficiency jumps from 3-25% to 95%+. This means 3-8x more concurrent requests on the same GPU.
</Callout>

## Page Size: The Only Tuning Knob

The page size controls the granularity of allocation. Smaller pages reduce internal fragmentation (wasted space within the last page) but increase page table overhead:

16 tokens per page is optimal for most workloads: it matches natural attention tile sizes in GPU kernels, keeps page table overhead at 2%, and limits internal waste to ~8 tokens per request.

## Copy-on-Write: Free Sharing

PagedAttention enables copy-on-write semantics for shared prefixes. When multiple requests share the same prompt (beam search, parallel sampling, shared system prompts), they reference the same physical pages. Pages are only copied when one request's decode diverges.

For beam search with width 4, this reduces KV memory from 4x to ~1.2x (the shared prefix is stored once, only diverging suffixes are duplicated).

<Callout type="info" title="Beyond Beam Search">
  Copy-on-write is increasingly important for prefix caching: if many users send the same system prompt, the KV cache for that prefix is computed once and shared across all requests. vLLM's automatic prefix caching leverages this, with PagedAttention making the sharing free at the memory level.
</Callout>

## Conclusion

PagedAttention eliminates KV cache memory waste by applying virtual memory concepts: fixed-size pages, per-request page tables, and on-demand allocation. The engineering insight is simple -- don't pre-allocate for maximum sequence length -- but the impact is transformative: 95%+ memory utilization enables 2-3x more concurrent requests and proportionally higher throughput. Combined with continuous batching, it's the foundation of modern LLM serving.
