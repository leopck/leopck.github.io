Distributed Data Parallel (DDP) is the foundation of all large-scale training. Each GPU has a complete copy of the model and processes a different mini-batch. After the backward pass, gradients must be synchronized so all GPUs have identical gradient values before the optimizer step. The synchronization method — bucket-based all-reduce overlapped with the backward pass — determines whether distributed training scales efficiently.

## The Core Mechanism

Each GPU runs forward + backward independently. After backward, each parameter has a local gradient. DDP synchronizes via all-reduce: compute the average gradient across all N GPUs.

$$g_{\text{synced}} = \frac{1}{N} \sum_{i=1}^{N} g_i$$

### Ring All-Reduce

The standard algorithm: arrange N GPUs in a ring. Each GPU sends a chunk to its neighbor and receives a chunk from the other neighbor. After $2(N-1)$ steps, all GPUs have the averaged gradient.

Communication volume per GPU: $2 \times \frac{(N-1)}{N} \times |\text{params}| \times \text{dtype\_bytes}$

For Llama 70B in FP32 gradients on 8 GPUs: $2 \times \frac{7}{8} \times 70\text{B} \times 4 = 490$ GB total communication per GPU. At NVLink 900 GB/s: 0.54 seconds. At InfiniBand 50 GB/s: 9.8 seconds.

## Bucket-Based Overlap

The key optimization: don't wait for the entire backward pass to finish before starting all-reduce. Instead, divide parameters into **buckets** (default 25MB each). As soon as a bucket's gradients are computed during backward, start its all-reduce immediately.

```python
# Simplified DDP backward with bucket overlap
class BucketDDP:
    def __init__(self, model, bucket_size_mb=25):
        self.model = model
        self.bucket_size = bucket_size_mb * 1024 * 1024  # bytes
        self.buckets = self._create_buckets()

    def _create_buckets(self):
        buckets = []
        current_bucket = []
        current_size = 0
        # Iterate parameters in REVERSE order (backward computes last layer first)
        for param in reversed(list(self.model.parameters())):
            param_size = param.numel() * param.element_size()
            if current_size + param_size > self.bucket_size and current_bucket:
                buckets.append(current_bucket)
                current_bucket = []
                current_size = 0
            current_bucket.append(param)
            current_size += param_size
        if current_bucket:
            buckets.append(current_bucket)
        return buckets

    def backward_with_sync(self, loss):
        # Register hooks: when a bucket's last param gets its gradient, start all-reduce
        bucket_events = {}
        for i, bucket in enumerate(self.buckets):
            last_param = bucket[-1]
            last_param.register_hook(
                lambda grad, bucket_id=i: self._all_reduce_bucket(bucket_id)
            )

        loss.backward()  # Triggers hooks as gradients are computed
        # All-reduce of early buckets overlaps with backward of later layers

    def _all_reduce_bucket(self, bucket_id):
        grads = [p.grad for p in self.buckets[bucket_id]]
        flat = torch.cat([g.flatten() for g in grads])
        torch.distributed.all_reduce(flat, op=torch.distributed.ReduceOp.AVG)
        # Unflatten back to individual gradients
        offset = 0
        for g in grads:
            numel = g.numel()
            g.copy_(flat[offset:offset+numel].view_as(g))
            offset += numel
```

<Callout type="perf" title="Overlap Hides 80-95% of Communication">
The backward pass takes ~3x the forward pass time (gradient computation through attention + FFN). All-reduce of early buckets (last layers) runs concurrently with backward computation of later buckets (earlier layers). For well-balanced models on NVLink, 80-95% of all-reduce time is hidden behind backward computation. The effective overhead: 5-20% of training step time, not the full 0.54 seconds.
</Callout>

## When DDP Is Not Enough

DDP requires each GPU to hold a complete copy of the model. For Llama 70B: 140 GB weights + 140 GB gradients + 560 GB optimizer states = 840 GB per GPU. No single GPU has this.

Solutions that build on DDP:
- **ZeRO-1**: Partition optimizer states. Each GPU stores 1/N of optimizer. All-reduce still needed for gradients. (Inference Timeline: DeepSpeed ZeRO post)
- **FSDP**: Partition everything — parameters, gradients, optimizer. All-gather parameters on demand. (Multi-GPU Parallelism post)
- **TP + DDP**: Split model across GPUs within a node (TP), replicate across nodes (DDP).
