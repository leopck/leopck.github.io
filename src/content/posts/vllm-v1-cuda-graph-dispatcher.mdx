CUDA graphs eliminate kernel launch overhead by recording a sequence of GPU operations and replaying them as a single unit. For LLM inference, this is critical: a single decode step for Llama 70B launches hundreds of kernels (attention, FFN, normalization, all-reduce for tensor parallelism), and the CPU-side launch overhead for each kernel is 5-10 microseconds. At 300+ kernels per step, that is 1.5-3 ms of pure launch overhead per token -- which dominates the total decode latency at small batch sizes.

The problem: CUDA graphs require fixed tensor shapes. LLM serving has dynamic batch sizes that change every iteration as requests arrive and complete. vLLM v1 solves this with a graph dispatcher that pads batch sizes to predetermined buckets, caches a graph per bucket, and dispatches to the correct graph at runtime.

This post covers the full dispatcher design: bucket selection strategy, graph capture pipeline, memory overhead of maintaining multiple graphs, integration with torch.compile, and the performance characteristics across batch sizes.

---

## 1. Why CUDA Graphs Matter for Decode

### The Launch Overhead Problem

Every CUDA kernel invocation requires the CPU to prepare launch parameters, push them onto the GPU command queue, and wait for the driver to schedule execution. This takes 5-10 microseconds per kernel on modern NVIDIA hardware. The GPU does not begin executing until the CPU finishes the launch sequence.

For a single Llama 70B decode step (one token, one batch element), the kernel count is approximately 885 kernels (11 per layer across 80 layers, plus final normalization, LM head, and sampling).

At 7 microseconds per launch: $885 \times 7\mu s = 6.2\text{ ms}$

The actual GPU compute time for a batch-1 decode step on an H100 is approximately 3-5 ms. So launch overhead exceeds the compute time.

### How CUDA Graphs Work

A CUDA graph is a DAG of GPU operations captured during a "recording" phase and replayed during an "execution" phase.

```python
import torch

# Capture phase
static_input = torch.randn(batch_size, hidden_dim, device="cuda")
static_output = torch.empty(batch_size, hidden_dim, device="cuda")

graph = torch.cuda.CUDAGraph()
with torch.cuda.graph(graph):
    static_output.copy_(model_forward(static_input))

# Replay phase
static_input.copy_(real_input)
graph.replay()
result = static_output.clone()
```

The constraint: every tensor in the graph has a fixed shape, dtype, and memory address. You cannot change the batch size after capture.

## 2. The Bucket Strategy

### Padding to Power-of-2

Since batch sizes change every iteration, vLLM pads the batch size up to the nearest "bucket" value and captures one graph per bucket.

```python
def get_graph_batch_size(batch_size: int) -> int:
    """Round up batch size to the nearest graph-capture bucket."""
    if batch_size <= 2:
        return 2
    elif batch_size <= 4:
        return 4
    else:
        return 1 << (batch_size - 1).bit_length()
```

The buckets are: {2, 4, 8, 16, 32, 64, 128, 256, ...} up to `max_num_seqs`. For a typical deployment with `max_num_seqs=256`, there are 8 buckets. The worst-case padding waste is 50% (batch of 33 padded to 64). On average, waste is 25%.

### Alternative Bucket Strategies

Some deployments use finer granularity at common batch sizes:

```python
def get_graph_batch_size_fine(batch_size: int) -> int:
    """Finer granularity buckets for production workloads."""
    fine_buckets = [1, 2, 4, 8, 12, 16, 24, 32, 48, 64, 96, 128, 192, 256]
    for bucket in fine_buckets:
        if batch_size <= bucket:
            return bucket
    return fine_buckets[-1]
```

<Callout type="perf" title="Bucket Selection Tradeoff">
More buckets = less padding waste = more memory. Fewer buckets = more padding waste = less memory. For most workloads, power-of-2 buckets are the right default. If your batch size distribution is concentrated around specific values, custom buckets can reduce waste by 30-40%.
</Callout>

## 3. The Graph Capture Pipeline

### Pre-Capture at Startup

vLLM captures all CUDA graphs during engine initialization, not at runtime:

```python
class CUDAGraphRunner:
    def __init__(self, model, max_batch_size: int):
        self.model = model
        self.graph_pool = {}
        self.input_buffers = {}
        self.output_buffers = {}

        bucket_sizes = self._compute_buckets(max_batch_size)
        for bs in reversed(bucket_sizes):
            self._capture_graph(bs)

    def _compute_buckets(self, max_bs: int):
        buckets = []
        bs = 2
        while bs <= max_bs:
            buckets.append(bs)
            bs *= 2
        return buckets

    def _capture_graph(self, batch_size: int):
        input_ids = torch.zeros(batch_size, dtype=torch.long, device="cuda")
        positions = torch.zeros(batch_size, dtype=torch.long, device="cuda")
        slot_mapping = torch.zeros(batch_size, dtype=torch.long, device="cuda")

        # Warmup
        with torch.no_grad():
            _ = self.model(input_ids, positions, slot_mapping)

        graph = torch.cuda.CUDAGraph()
        with torch.cuda.graph(graph, pool=self._get_memory_pool()):
            output = self.model(input_ids, positions, slot_mapping)

        self.graph_pool[batch_size] = graph
        self.input_buffers[batch_size] = {
            "input_ids": input_ids,
            "positions": positions,
            "slot_mapping": slot_mapping,
        }
        self.output_buffers[batch_size] = output
```

The `reversed(bucket_sizes)` ordering captures the largest batch size first, ensuring the CUDA memory allocator reserves enough memory.

### Memory Pool Sharing

Without a shared pool, each graph allocates its own intermediates, causing massive memory waste:

```python
def _get_memory_pool(self):
    if not hasattr(self, "_pool"):
        self._pool = torch.cuda.graph_pool_handle()
    return self._pool
```

With pool sharing, all graphs share the same intermediate memory region. Since only one graph executes at a time, this is safe.

## 4. The Dispatch Path

### Runtime Graph Selection

At runtime, the dispatcher selects the correct graph, copies real data into static buffers, replays, and extracts output:

```python
class CUDAGraphDispatcher:
    def execute(self, input_ids, positions, slot_mapping):
        real_batch_size = input_ids.shape[0]
        graph_batch_size = get_graph_batch_size(real_batch_size)

        if graph_batch_size not in self.runner.graph_pool:
            return self.runner.model(input_ids, positions, slot_mapping)

        buffers = self.runner.input_buffers[graph_batch_size]
        graph = self.runner.graph_pool[graph_batch_size]

        buffers["input_ids"][:real_batch_size].copy_(input_ids)
        buffers["positions"][:real_batch_size].copy_(positions)
        buffers["slot_mapping"][:real_batch_size].copy_(slot_mapping)

        if real_batch_size < graph_batch_size:
            buffers["input_ids"][real_batch_size:].zero_()
            buffers["positions"][real_batch_size:].zero_()
            buffers["slot_mapping"][real_batch_size:].fill_(-1)

        graph.replay()
        output = self.runner.output_buffers[graph_batch_size]
        return output[:real_batch_size]
```

The dispatch overhead is minimal: three `copy_` operations and one `graph.replay()`.

### Slot Mapping for Padded Positions

For padded positions, the slot mapping must point to a "null" location:

NULL_BLOCK_ID = 0

## 5. torch.compile Integration

### The Compilation Pipeline

vLLM v1 integrates `torch.compile` with CUDA graphs in a two-stage pipeline:

```python
# Stage 1: Compile the model
compiled_model = torch.compile(
    model,
    mode="max-autotune",
    fullgraph=True,       # No graph breaks allowed
    dynamic=False,        # Static shapes for CUDA graphs
)

# Stage 2: Capture CUDA graphs of the compiled model
runner = CUDAGraphRunner(compiled_model, max_batch_size=256)
```

The `fullgraph=True` flag is critical. Graph breaks fall back to eager execution, which breaks CUDA graph capture. Common sources of graph breaks:

```python
# These cause graph breaks:
if tensor.item() > 0:          # Data-dependent control flow
print(tensor)                   # Python side effects
tensor.numpy()                  # Transfer to CPU

# These are safe:
torch.where(cond, a, b)        # Branchless conditional
torch.clamp(tensor, min=0)     # Pure tensor ops
```

## 6. Prefill vs. Decode: When Graphs Apply

vLLM v1 uses CUDA graphs exclusively for decode. Prefill has variable sequence lengths, making graph capture impractical.

```python
class ModelRunner:
    def execute_model(self, input_data):
        if input_data.is_prefill:
            return self.compiled_model(
                input_data.input_ids, input_data.positions,
                input_data.attn_metadata,
            )
        else:
            return self.graph_dispatcher.execute(
                input_data.input_ids, input_data.positions,
                input_data.slot_mapping,
            )
```

With chunked prefill, the boundary blurs. vLLM v1 uses graphs only for pure-decode batches:

```python
def should_use_graphs(batch):
    return (
        batch.num_prefill_tokens == 0
        and batch.num_decode_tokens > 0
        and batch.num_decode_tokens <= max_graph_batch_size
    )
```

## 7. Memory Overhead Analysis

Each captured graph requires static input and output buffers. For Llama 70B ($d_{\text{model}} = 8192$, FP16):

$$\text{buffer\_bytes}(bs) = bs \times (8 + 8 + 8 + 8192 \times 2) = bs \times 16408 \text{ bytes}$$

Total across all buckets (2 through 256): approximately 8.4 MB for static buffers (negligible). The real cost is CUDA graph internal state -- approximately 3.6 GB with memory pool sharing.

<Callout type="warning" title="Graph Memory Reduces KV Cache Capacity">
The memory consumed by CUDA graphs directly reduces the amount of HBM available for KV cache blocks. With 8 buckets up to bs=256, approximately 3.6 GB of HBM is reserved for CUDA graphs. On an 80 GB H100, this reduces KV cache capacity by about 4.5%.
</Callout>

### Memory Budget Calculator

```python
def compute_graph_memory(
    model_hidden_dim: int,
    max_batch_size: int,
    num_layers: int,
    dtype_bytes: int = 2,
):
    buckets = []
    bs = 2
    while bs <= max_batch_size:
        buckets.append(bs)
        bs *= 2

    static_per_element = 8 + 8 + 8 + model_hidden_dim * dtype_bytes
    total_static = sum(bs * static_per_element for bs in buckets)
    intermediate = max_batch_size * model_hidden_dim * dtype_bytes * 2 * num_layers
    metadata = len(buckets) * 50 * 1024

    return {
        "num_buckets": len(buckets),
        "buckets": buckets,
        "static_buffers_mb": total_static / (1024 ** 2),
        "intermediate_mb": intermediate / (1024 ** 2),
        "total_mb": (total_static + intermediate + metadata) / (1024 ** 2),
    }
```

## 8. Dynamic Graph Capture: Lazy vs. Eager

### Lazy Capture (Alternative)

An alternative captures graphs on-demand the first time each bucket is needed:

```python
class LazyCUDAGraphRunner:
    def __init__(self, model, max_batch_size):
        self.model = model
        self.max_batch_size = max_batch_size
        self.graph_pool = {}
        self.input_buffers = {}
        self.output_buffers = {}
        self._pool = torch.cuda.graph_pool_handle()

    def execute(self, input_ids, positions, slot_mapping):
        bs = get_graph_batch_size(input_ids.shape[0])
        if bs not in self.graph_pool:
            self._capture_graph(bs)

        buffers = self.input_buffers[bs]
        real_bs = input_ids.shape[0]

        buffers["input_ids"][:real_bs].copy_(input_ids)
        buffers["positions"][:real_bs].copy_(positions)
        buffers["slot_mapping"][:real_bs].copy_(slot_mapping)

        if real_bs < bs:
            buffers["input_ids"][real_bs:].zero_()
            buffers["slot_mapping"][real_bs:].fill_(-1)

        self.graph_pool[bs].replay()
        return self.output_buffers[bs][:real_bs]
```

Lazy capture has faster startup but adds latency to the first request in each bucket. For production, eager capture is preferred.

## 9. Multi-GPU Considerations

### Tensor Parallelism and CUDA Graphs

With TP, each GPU captures its own graph including NCCL all-reduce operations:

```python
def capture_tp_graphs(model, tp_group, max_batch_size):
    buckets = compute_buckets(max_batch_size)
    for bs in reversed(buckets):
        torch.distributed.barrier(group=tp_group)
        graph = torch.cuda.CUDAGraph()
        with torch.cuda.graph(graph, pool=get_pool()):
            output = model.forward_shard(
                input_ids=static_input[bs],
                positions=static_positions[bs],
            )
        graphs[bs] = graph
        torch.distributed.barrier(group=tp_group)
```

Pipeline parallelism involves conditional control flow between stages and does not support CUDA graphs.

## 10. Debugging CUDA Graph Issues

### Common Failure Modes

**Shape mismatch**: Input tensor shape does not match captured shape. Fix: always pad to bucket size.

**Memory allocation during replay**: Model allocates a new tensor during forward pass. Fix: pre-allocate all tensors; use `fullgraph=True`.

**CPU-side operations in graph**: Operations like `.item()`, `.cpu()`, `print()` inside captured region. Fix: remove all CPU-side operations from the forward path.

### Diagnostic Tool

```python
def validate_graph_capture(model, batch_size):
    input_ids = torch.randint(0, 32000, (batch_size,), device="cuda")
    positions = torch.arange(batch_size, device="cuda")
    slot_mapping = torch.arange(batch_size, device="cuda")

    with torch.no_grad():
        eager_out = model(input_ids, positions, slot_mapping)

    static_ids = torch.zeros(batch_size, dtype=torch.long, device="cuda")
    static_pos = torch.zeros(batch_size, dtype=torch.long, device="cuda")
    static_slots = torch.zeros(batch_size, dtype=torch.long, device="cuda")

    graph = torch.cuda.CUDAGraph()
    with torch.cuda.graph(graph):
        graph_out = model(static_ids, static_pos, static_slots)

    static_ids.copy_(input_ids)
    static_pos.copy_(positions)
    static_slots.copy_(slot_mapping)
    graph.replay()

    max_diff = (eager_out - graph_out).abs().max().item()
    assert max_diff < 1e-3, f"Graph output diverges: {max_diff}"
    return True
```

## 11. Complete Minimal Implementation

```python
import torch
from typing import Optional

class MinimalGraphDispatcher:
    """
    Minimal CUDA graph dispatcher for LLM decode.
    Demonstrates: bucket selection, graph capture, memory pooling, dispatch.
    """

    def __init__(self, model: torch.nn.Module, max_batch_size: int = 256):
        self.model = model
        self.max_batch_size = max_batch_size
        self._pool = torch.cuda.graph_pool_handle()
        self._graphs = {}
        self._input_buffers = {}
        self._output_buffers = {}

        for bs in self._get_buckets():
            self._capture(bs)

    def _get_buckets(self):
        buckets = []
        bs = 2
        while bs <= self.max_batch_size:
            buckets.append(bs)
            bs *= 2
        return list(reversed(buckets))

    @staticmethod
    def _pad_to_bucket(batch_size: int) -> int:
        if batch_size <= 2:
            return 2
        return 1 << (batch_size - 1).bit_length()

    def _capture(self, batch_size: int):
        ids = torch.zeros(batch_size, dtype=torch.long, device="cuda")
        pos = torch.arange(batch_size, device="cuda")

        with torch.no_grad():
            _ = self.model(ids, pos)

        g = torch.cuda.CUDAGraph()
        with torch.cuda.graph(g, pool=self._pool):
            out = self.model(ids, pos)

        self._graphs[batch_size] = g
        self._input_buffers[batch_size] = (ids, pos)
        self._output_buffers[batch_size] = out

    def __call__(self, input_ids: torch.Tensor, positions: torch.Tensor) -> torch.Tensor:
        real_bs = input_ids.shape[0]
        padded_bs = self._pad_to_bucket(real_bs)

        if padded_bs > self.max_batch_size or padded_bs not in self._graphs:
            return self.model(input_ids, positions)

        ids_buf, pos_buf = self._input_buffers[padded_bs]
        ids_buf[:real_bs].copy_(input_ids)
        pos_buf[:real_bs].copy_(positions)

        if real_bs < padded_bs:
            ids_buf[real_bs:].zero_()
            pos_buf[real_bs:].fill_(0)

        self._graphs[padded_bs].replay()
        return self._output_buffers[padded_bs][:real_bs].clone()
```
